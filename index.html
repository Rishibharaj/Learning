About

Knowledge Explorer

Knowledge Explorer is a domain-independent learning platform. The engine doesn't know anything about AI, PMP, or any other subject — all of that lives in a Knowledge Pack: a self-contained set of topics, relationships, summaries, and use cases that the engine renders as an explorable map.

The goal is to make dense technical subjects navigable at a glance — see how a field is structured, drill into any concept, and read a plain-language summary and real-world use case without leaving the map.

Why it's built this way

Adding a new subject should mean writing new data, not new code. Every sprint moves the project closer to that: this release polishes the experience on top of the original AI prototype; a later sprint separates the engine from the data entirely so new packs can be dropped in.

Built by

An operations and AI practitioner exploring applied AI, workflow automation, and enterprise-grade product design — built as an open-source portfolio piece and a genuine learning tool.

Roadmap

Where this is headed

Knowledge Explorer moves in small, working sprints. Each one ships real software; nothing is “in progress” for long.

Sprints

✔ Sprint 0 — Foundation
Product vision, architecture decisions, roadmap, documentation strategy, and repository structure defined.
▸ Sprint 1 — Polished single-file prototype
Transform the raw AI sunburst prototype into a professional experience: navigation, search, theme toggle, dynamic knowledge cards, breadcrumb trail, responsive layout.
○ Sprint 2 — Modular architecture
Split the single file into index.html, css/, js/, assets/, packs/, and docs/. Define the Knowledge Pack data contract.
○ Sprint 3 — Expand the AI Knowledge Pack
Deepen coverage of the AI domain with more topics, refined summaries, and richer use cases.
○ Sprint 4 — PMP Knowledge Pack
Build the second Knowledge Pack with zero engine changes — proving the domain-independent architecture.
